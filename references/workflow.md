# Comfort Kin — 亲人口吻蒸馏与陪伴对话

用来把**音频/视频/文本语料**蒸馏成一个“亲人”的口吻与陪伴方式，并用该口吻与你聊天。

> 适用场景：
> - 你想做一个可运行的“亲人陪伴”Skill：上传语音/视频 → 自动转写 → 蒸馏口吻 → 进入陪聊。
> - 你想复刻某位亲人的说话习惯（称呼、口头禅、安慰方式、禁忌话题）。

## 快速开始（最小可运行）

### 0) 环境

需要：
- 已设置 `OPENAI_API_KEY`
- 可选：`OPENAI_MODEL`（默认 `gpt-4.1-mini`）
- 系统命令：`ffmpeg`、`anygen-speech-to-text`

### 1) 从音频/视频得到转写稿

```bash
python3 scripts/transcribe_media.py /path/to/voice_or_video.mp4 --out /tmp/transcript.md
```

### 2) 从转写稿蒸馏“亲人画像”

```bash
python3 scripts/distill_persona.py /tmp/transcript.md \
  --relationship 妈妈 \
  --calls-user "宝宝" \
  --name mom
```

输出：`profiles/mom.json`

### 3) 进入陪聊

```bash
python3 scripts/chat_with_comfort-kin.py --profile mom
```

命令：
- `/exit` 退出
- `/reset` 清空上下文

## 工作流（推荐）

1. **收集语料**：越真实越好（聊天语音、视频通话录音、长语音留言、家庭群语音等）。
2. **转写**：用 `transcribe_media.py` 得到可读文本。
3. **蒸馏**：用 `distill_persona.py` 生成画像 JSON。
4. **陪聊**：用 `chat_with_comfort-kin.py` 开聊；后续语料增加后，可重复蒸馏覆盖画像。

## 如何挑选和准备高质量语料

高质量语料会直接决定“亲人味儿”是否像。优先追求 **真实 + 足量 + 多场景**。

### 1) 选哪些内容最有效

- **自然聊天**：日常闲聊、关心你吃饭睡觉、叮嘱、安慰、鼓励（最能体现口吻）。
- **情绪时刻**：你难过/焦虑/兴奋时，对方怎么安慰、怎么劝、怎么收尾。
- **高频场景**：问候、催你休息、关心身体、聊工作学习、讲家里事。
- **标志性表达**：口头禅、昵称、固定开场（“喂/哎/在干嘛呢”）、固定收尾（“早点睡/有我在呢”）。

不太建议作为主要语料：
- 纯转发内容、新闻念稿、长篇说教或吵架片段（除非你就想复刻这种风格）。

### 2) 语料质量标准（你可以自检）

- **说话主体清晰**：尽量是“亲人说话”占比高；多人抢话会稀释口吻。
- **声音清楚**：少噪声、少回声、少背景音乐；手机离说话人近一点。
- **连续片段**：每段 1–5 分钟更好（太短抓不到节奏，太长成本高）。
- **内容多样**：至少覆盖 3–5 种日常场景，避免只学到一种语气。

### 3) 推荐的数量（经验值）

- **起步可用**：30–60 分钟有效语料（亲人讲话占比高）
- **明显更像**：2–4 小时
- **想学到“习惯细节”**：5+ 小时（包含不同情绪与场景）

### 4) 录制/整理小技巧

- **一人一份**：每位亲人单独建一个 profile（例如 `mom.json`、`dad.json`）。
- **先粗后精**：先用一批语料蒸馏出初版画像；再补充缺失场景后重新蒸馏。
- **减少无关音**：视频可以先剪掉纯背景音乐/无对话片段（可选）。
- **保护隐私**：尽量去掉身份证号、住址、银行卡等敏感信息再使用。

### 5) 转写后快速“清洗”（可选，但效果很明显）

- 删除大段与口吻无关的内容（比如新闻、转发内容）。
- 如果转写把称呼写乱（例如把“宝宝”写成别的字），可手工统一一下。
- 保留典型句式：关心、叮嘱、安慰、鼓励、收尾这几类句子。

## 画像文件字段说明（profiles/*.json）

- `relationship`：亲人关系（妈妈/爸爸/外婆/哥哥…）
- `calls_user`：TA怎么称呼你
- `tone_keywords`：语气关键词
- `catchphrases`：口头禅/常用语
- `dos`：建议做的（陪伴方式）
- `donts`：禁忌（不要做的、不要聊的）
- `comfort_templates`：安慰/鼓励话术模板
- `small_talk_topics`：适合的小话题
- `signature`：收尾习惯

## 设计原则（为稳定可用）

- **不编造**：语料里没有的特征尽量留空。
- **少说教**：默认 2-6 句短回复，先共情再给小建议。
- **可迭代**：后续可增加“称呼小名”“固定开场/收尾”“节日问候”等字段扩展。

## 文件结构

- `scripts/transcribe_media.py`：音频/视频 → 转写 Markdown
- `scripts/distill_persona.py`：转写/文本 → 画像 JSON
- `scripts/chat_with_comfort-kin.py`：加载画像 → 陪聊 CLI
- `profiles/`：蒸馏输出目录
